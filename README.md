# 리눅스 명령어 조사: top, ps, jobs, kill

## 1. top 명령어
`top` 명령어는 리눅스 시스템에서 실행 중인 프로세스와 시스템의 자원 사용 현황을 실시간으로 보여주는 강력한 유틸리티이다.

### 주요 기능
- 실시간 모니터링: 일정 주기로 화면을 업데이트하여 시스템 성능을 실시간으로 모니터링할 수 있다.
- 프로세스 정보: PID, 사용자, 우선순위, 메모리 사용량, CPU 사용량 등을 표시한다.
- 시스템 요약: 가동 시간, 사용자 수, 로드 평균, CPU 상태, 메모리/스왑 사용량 등을 요약하여 보여준다.

### 자주 쓰는 옵션
| 옵션 | 설명 |
|------|------|
| `-u 사용자명` | 특정 사용자 프로세스만 보기 |
| `-p PID` | 특정 PID만 모니터링 |
| `-n N` | 화면 갱신 횟수 지정 |

---

## 2. ps 명령어
`ps` 명령어는 Process Status의 약자로, 현재 실행 중인 프로세스의 정보를 출력하는 명령어이다.

### 자주 쓰는 옵션
| 옵션 | 설명 |
|------|------|
| `-A` | 모든 프로세스를 출력 |
| `-u username` | 특정 사용자가 실행한 프로세스 표시 |
| `-ef --forest` | 부모-자식 관계를 트리 구조로 표현 |

---

## 3. jobs 명령어
`jobs` 명령어는 현재 쉘 세션에서 실행 중인 백그라운드 및 중지된 작업 목록을 확인하는 명령어이다.  
작업 번호, 실행 상태 등을 확인할 수 있어 작업 제어에 사용된다.

### 출력되는 상태값
| 상태 | 설명 |
|------|------|
| `Running` | 작업이 실행 중 |
| `Done` | 작업이 정상 종료됨 |
| `Done(code)` | 비정상 종료(Code 반환) |
| `Stopped` | Ctrl+Z 등으로 일시 중단 |

---

## 4. kill 명령어
`kill` 명령어는 특정 프로세스(PID)에 시그널(signal)을 보내 종료하거나 제어하는 명령어이다.

### 자주 쓰는 옵션
| 옵션 | 설명 |
|------|------|
| `-s signal` | 특정 시그널을 지정하여 전송 |
| `-l` | 사용 가능한 시그널 목록 출력 |
| `-9` | SIGKILL 강제 종료 |
| `-15` | SIGTERM 정상 종료 요청(기본값) |

### 사용 예시
```bash
kill -9 1234
```
